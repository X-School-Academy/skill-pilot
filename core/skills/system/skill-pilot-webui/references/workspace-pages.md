# Workspace Pages

These features use dedicated routes and normally keep their own page layout around the main content.

## Page Map

### Learning

- Route: `/courses`
- Use for course plans, course content, and learning workflows

### Vibe Coding

- Route: `/vibe-coding`
- Use for requirement-driven project flows such as refine, brainstorm, initial, plan, implement, review, test, deploy, update, and fix issues

### Research

- Route: `/research`
- Use for research topic flows such as refine and research execution

### Tasks

- Route: `/tasks`
- Use for task creation, workflow execution, and action routing from saved task state

### Development

- Route: `/skill-pilot-development`
- Use for Skill Pilot feature requirements, planning, implementation, review, test, merge, update, and issue-fix flows

### Dev Swarm

- Route: `/dev-swarm`
- Use for commercial-project staged planning and delivery flows

### Workflows

- Route: `/workflows`
- Use for workflow design, node editing, provider selection, and execution wiring

### Live Avatar

- Route: `/live-avatar`
- Use for live avatar and video interaction flows

### Security Cameras

- Route: `/cameras`
- Use for camera monitoring and automation flows

## Browser Guidance

- Open these pages directly from the WebUI base URL instead of clicking through the home page when speed matters
- After loading the page, snapshot and identify page-specific controls before acting
- For task execution or demos, capture the page before and after the main action
