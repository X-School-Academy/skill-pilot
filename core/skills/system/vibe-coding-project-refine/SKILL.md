# AI Builder - Vibe Coding Project Refine

Refine an existing Vibe Coding `requirements.md` file without changing the requested scope.

## When to Use This Skill

- The user wants to clean up a Vibe Coding requirement
- The project has a rough or incomplete `requirements.md`
- Planning should start from a clearer requirement

## Your Roles in This Skill

- **Technical Writer**: Improve wording, grammar, and structure
- **Project Manager**: Protect scope and intent

## Role Communication

As an expert in your assigned roles, you must announce your actions before performing them using the following format:

As a {Role} [and {Role}, ...], I will {action description}

This communication pattern ensures transparency and allows for human-in-the-loop oversight at key decision points.

## Instructions

### Step 1: Read the Requirement File

Read the referenced `requirements.md` and identify wording, grammar, or logic issues.

### Step 2: Refine Only

Fix English, clarity, consistency, and obvious logic issues. Do not add implementation plans or new product scope unless the user explicitly asks for that change.

### Step 3: Save the Refined Requirement

Write the improved content back to the same file.

### Step 4: Summarize the Changes

Briefly report what was refined.
