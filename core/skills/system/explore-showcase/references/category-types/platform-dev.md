# Category: Platform Dev — Types

Improve Skill Pilot itself: codeware status, worktrees, features, contributions.

Showcases in this category typically use `in_mode: dev` so changes are observable in the dev WebUI with live-reload.

Primary audiences: platform contributors, advanced AI agent builders.

## Types

### PD1. Codeware sync / restore
- Run `codeware` checks, restore a clean baseline, configure a personal fork.

### PD2. Add a new feature
- Drive the full lifecycle: create → refine → plan → implement → review → test → merge.

### PD3. Fix a bug
- Reproduce, diagnose, write a failing test, fix, regression test, ship.

### PD4. Reverse engineer a feature
- Remove an existing feature, draft `requirements.md`, set `git_tag` + `use_worktree: true` + `in_mode: dev`, ask the user to re-implement it.

### PD5. UI / extension update
- Add or update a WebUI extension; verify live-reload behavior.

### PD6. Make an upstream contribution
- Branch, format, lint, write a conventional commit, open a PR through `git-github`.

### PD7. Documentation / licensing pass
- Update docs, README, license/compliance pages.
