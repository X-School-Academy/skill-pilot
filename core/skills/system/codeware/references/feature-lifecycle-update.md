# Feature Lifecycle: Update

Use when an existing feature needs additional changes. This stage sets up the trigger doc only — planning and implementation follow as separate stages.

## Steps

1. If `update.md` already exists from a previous cycle, archive it as `archive/update.{timestamp}.md`. Create the `archive/` folder if it does not exist.
2. If `update.md` does not exist, create it from the user's description of the requested changes.
3. Proceed to `refine` → `initial` → `plan` → `implement` → `test` → `review` → `merge`.
