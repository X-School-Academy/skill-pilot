# Feature Lifecycle: Fix Issues

Use when a feature has bug reports or issues to resolve. This stage sets up the trigger doc only — planning and implementation follow as separate stages.

## Steps

1. If `issues.md` already exists from a previous cycle, archive it as `archive/issues.{timestamp}.md`. Create the `archive/` folder if it does not exist.
2. If `issues.md` does not exist, create it from the user's bug descriptions or issue notes.
3. Proceed to `refine` → `initial` → `plan` → `implement` → `test` → `review` → `merge`.
