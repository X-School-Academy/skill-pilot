# Feature Lifecycle: Review

Use when a feature implementation should be audited before merge.

## Steps

1. Read the relevant implementation context (`implementation.md`, `update-impl.md`, or `issues-impl.md`) and inspect the code.
2. Identify all issues, ordered by severity, with file and line references. Focus on bugs, regressions, missing tests, and feature-scope risks.
3. Fix every issue found before reporting done.
4. Confirm each fix is in place, then report the review as complete.
