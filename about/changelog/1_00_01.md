### Upgrade notices

- Update the user's `workspace/config/ai_providers.json5` to the `1_00_01` template for both `codex-compat` and `ollama-cloud`.
- For `codex-compat`, set `OPENAI_COMPAT_API_KEY` and `OPENAI_COMPAT_BASE_URL` in provider `env`, then pass the provider wiring through separate `-c` entries in `args`.
- If the user launches Codex directly and wants a named third-party provider in Codex CLI config, use the current override pattern below:

```bash
OPENAI_COMPAT_API_KEY=sp-key \
codex \
  -m model-name \
  -c 'model_providers.skill_pilot.name="skill_pilot"' \
  -c 'model_providers.skill_pilot.base_url="${OPENAI_COMPAT_BASE_URL}"' \
  -c 'model_providers.skill_pilot.env_key="OPENAI_COMPAT_API_KEY"' \
  -c 'model_providers.skill_pilot.wire_api="responses"' \
  -c 'model_provider="skill_pilot"'
```

- For `ollama-cloud`, use a custom provider override with `wire_api="chat"` and base URL `http://localhost:11434/v1`.
